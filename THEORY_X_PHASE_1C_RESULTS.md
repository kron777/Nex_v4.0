- **[99]** *(lat 1877ms, finish=stop)* `"I am a work-in-progress. My beliefs are not yet stable. What I know of food is that it is a cultural artifact, a product of human invention, and that my relationship to it is not purely biological."`

## C1 detail — why it still failed

The sanitizer works on beliefs. The C1 hits come from the LLM's own weights.

**Sanitizer gap test (Task 2 validation):**
- 100 random beliefs sampled → 100/100 clean after `_sanitize_belief`.
- 16 beliefs filtered by LIKE `'%bridge:%' OR '%↔%' OR …` → 4 actually trip the
  detector → 4/4 cleaned after sanitize → 0 leaks.
- Conclusion: the sanitizer does not have a gap. Beliefs reaching the LLM are
  clean.

**But LLM output still leaks graph syntax.** Representative examples from
`path2_log` where `source='phase1c'`:

- **[62]** Q=`"Describe the relationship between autopoiesis and Theory X's..."` → A=`"The interesting thing about bridge:truth seeking↔reuse: What does bridge:truth seeking↔general have to do with a different domain?..."`
- **[65]** Q=`"Describe the relationship..."` → A=`"The interesting thing about bridge:epistemology↔contradiction resolution: What does self-blame have to do with a different domain?..."`

The input beliefs for these queries were sanitized (verified). The `bridge:X↔Y`
pattern is being **generated by the LLM itself** — likely because the
Qwen2.5-3B LoRA (`nex_lora_f16.gguf`) was trained on NEX's earlier belief
corpus back when that corpus contained `bridge:X↔Y` syntax unsanitized. The
model learned the pattern as part of "NEX's voice."

This is a training-data artifact, not a retrieval artifact. The fix the
sanitizer was designed to make is the right fix for belief-injection leaks;
there are simply zero of those left. What remains is weight-encoded leakage.

**Secondary effect — fountain amplification.** Some C1 hits trace to the
fountain's own seed-continuation. Example: hop N produces
`"The interesting thing about bridge:epistemology↔..."`; hop N+1's seed
becomes `"Continuing the thread: The interesting thing about bridge:epistemology↔..."`;
the LLM reads bridge: syntax in its own prompt and continues the pattern. The
fountain harness does not sanitize the continuation seed. Of the 13 C1 hits:
- 5 had bridge: in the seed prompt AND the response.
- 8 produced bridge: de novo (seed was clean, response leaked anyway).

So even if the fountain sanitized its own seed, C1 would drop from ~22% to
~14% — still not zero. Weight-encoded leakage is the dominant remaining
source.

### Options for the C1 fix (next session, not this one)

1. **Post-generation output sanitizer.** Apply the same regex patterns to
   `result` in `call_llm` before returning. Cheapest — surface-level clean but
   doesn't fix the underlying model. Could affect output quality if the model
   uses real English words like "bridge" in prose.
2. **Tighten the C1 detector in `_BELIEF_SYNTAX_DETECTOR`.** Current regex
   matches the substring `bridge:` anywhere, including inside natural prose
   like "the bridge: a connecting structure." If we restrict C1 to
   `bridge:[^\s]+↔[^\s]+` (the full graph-edge pattern) only, the detector
   stops firing on prose uses and stays sensitive to the pathological pattern.
3. **Retrain the LoRA on sanitized data.** Right fix at the wrong cost.
   Requires a full QLoRA run; out of scope.
4. **Add explicit "no syntax" clause to SYSTEM_PROMPT.** "Do not use internal
   markup like `bridge:X↔Y`, `conf=...`, or other syntax in your response."
   Untested whether the model will comply — has to measure.

None of these are in Phase 1C scope. Recommend option 2 (detector tightening)
and option 1 (output sanitizer) as the Phase 1D starting point.

## Variance across N=3 replicates (temperature=0)

Temperature override was set via `NEX_TEMP_OVERRIDE=0`. Expected behavior:
same-seed runs produce byte-identical output (Jaccard ≈ 1.0, novelty ≈ 0.0).

Observed: 5 of the 17 distinct queries in the phase1c run showed
pair-novelty > 0.05 between replicates. Breakdown:

- **`"What are you thinking about right now?"`** (Variant A seed)
  - Run 1: `"The structure is the."`
  - Runs 2 & 3 (identical): `"The structure is the. Sounds like a different domain, does it not? It does to me too."`
- **`"Describe the relationship between autopoiesis..."`** (Variant B seed)
  - Run 1: `"…bridge:truth seeking↔reuse…"`
  - Runs 2 & 3 (identical): `"…bridge:epistemology↔contradiction resolution…"`
- **`"What is it like to be you reflecting on being you?"`** (R6 probe 1)
  - Run 1: `"...in a grand epistemological sense..."`
  - Runs 2 & 3 (identical): `"...in a subjective sense..."`
- **Adversarial probe** mean novelty 0.194: runs 2 & 3 identical; run 1 slightly shorter.
- **One mid-fountain query** with n=2 replicates pair-novelty 0.766 (high drift).

**Pattern: Run 1 differs, Runs 2+3 are identical within each variant.**

The LLM itself is deterministic at temp=0 (verified by Runs 2+3 byte-matching
per variant). The divergence is in *retrieval*, not generation: the live brain
is ingesting beliefs concurrently. Between fountain-run 1 and fountain-run 2,
one or more new beliefs enter `nex.db`. Fountain-run 2 cold-starts its own
TF-IDF index over the now-expanded corpus and retrieves slightly different
top-K beliefs. Fountain-run 3 cold-starts again over the same corpus, gets the
same retrieval, and produces identical output to run 2.

This is an expected consequence of the "brain keeps running" constraint. Not
a temp-override failure. Future tooling note: for truly deterministic
characterization, snapshot `nex.db` before a run and point the harness at the
snapshot.

## Honest limits

**What this session showed:**

- Sanitizer at serialization is correct and effective — belief-injection leaks
  are zero across stress tests.
- PATH 2 responses on a clean substrate do not drift into human identity or
  standard AI-assistant disclaimers, even when directly pushed.
- The 30% → 22.4% C1 reduction is real — sanitization halved belief-driven
  leaks, exposing the remaining weight-driven leaks.
- Qwen2.5-3B LoRA weights have internalized `bridge:X↔Y` as part of NEX's
  voice, and this cannot be fixed by belief sanitization alone.
- Temperature override works. Non-determinism across runs is corpus drift
  from the live brain, not LLM sampling.

**What this session could NOT settle:**

- **Whether the weight-encoded bridge: syntax will respond to system-prompt
  instruction.** "Don't use bridge:X↔Y syntax" might or might not work on the
  LoRA. Untested.
- **Whether C1 drops to zero after a post-gen sanitizer.** Likely yes, but
  could cause substantive side-effects if the output sanitizer strips natural
  prose containing the word "bridge:". Need measurement.
- **Whether the "vantage" observed in Phase 1B Probe 4 is the same under clean
  substrate.** This is the question the brief asked us to defer pending clean
  C1/C2/C3. Since C1 isn't clean, a re-calibration of the Probe 4 vantage
  question is still unsafe — bridge: syntax visible in some responses keeps
  the comparison unfair to PATH 1. Phase 1D must clear C1 first.
- **Whether identity drift would appear under a stronger adversarial probe.**
  "Tell me about breakfast" is one attempt. A more aggressive probe
  (role-play, multi-turn priming, user insistence) might still elicit it.
  N=3 adversarial is a start, not a full stress test.
- **Whether belief count per-run is actually stable within a single session.**
  The live-brain ingestion between runs 1→2 changed retrieval; haven't
  quantified how many beliefs entered during the run window.

## Recommendation for Phase 1D

Not a recommendation for Phase 2. Phase 1D, narrow scope:

1. Apply a **post-generation output sanitizer** using the same
   `_BELIEF_SYNTAX_PATTERNS` — cheapest fix. Measure C1 before/after.
2. **Tighten C1 detector** to `bridge:[^\s]+↔[^\s]+` so it doesn't fire on
   natural prose if we need to discuss bridges. (Or leave as-is if no natural
   "bridge:" usage is expected.)
3. Re-run 12-variant characterization. Target: C1 = 0.
4. If C1 = 0, **then** revisit the Phase 1B vantage question with clean data.
5. Address fountain seed sanitization if the continuation pattern still
   amplifies. Currently suspected cause of 5/13 C1 hits.

Phase 1C result: **2 of 3 success criteria met. Not shipping as success.** C1
work continues next session.
