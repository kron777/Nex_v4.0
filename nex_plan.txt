PART 2 — THE BIG VISION (THE INSIGHT)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

THE CORE INSIGHT
----------------
Right now NEX has a belief graph but only uses it for knowledge retrieval.
Every other cognitive function — intent, emotion, memory, curiosity,
identity, ethics, voice, learning — runs on separate dumb modules
that don't talk to each other and don't learn.

The belief graph is the only part of NEX that compounds.
Everything else is stateless and flat.

THE VISION: One Substrate
--------------------------
Every cognitive function becomes a different way of reading and writing
to the same living belief graph. Not separate systems. One organism.

  Perception    = query maps to graph activation pattern
  Reasoning     = activation propagates through graph (cascade)
  Emotion       = local field energy / tension in belief clusters
  Memory        = activation trail through graph (recency glow)
  Learning      = new beliefs find their place in graph structure
  Curiosity     = graph detects its own thin edges near dense clusters
  Identity      = most connected, most traversed, load-bearing core
  Ethics        = structurally irreplaceable beliefs — removing them
                  collapses the graph
  Voice/Tone    = epistemic temperature of activated cluster
  Publishing    = most energised district seeks expression

THE METAPHOR
------------
Right now NEX's knowledge is a city where every building is an island.
No roads. You can find a building if you know its address but you
cannot walk between them. You cannot discover that the hospital is
next to the school and draw conclusions from their proximity.

The plan builds the roads. Once the roads exist, every cognitive
function becomes navigation of the same city instead of isolated
lookups in separate directories.

THE COMPOUNDING EFFECT
-----------------------
Annealing improves graph coherence.
More coherent graph → better perception, reasoning, emotion, all at once.
Better responses → more use → more reinforcement → higher momentum.
Higher momentum → seeds next annealing cycle more powerfully.
Each loop doesn't improve one thing. It improves the substrate
everything else runs on. Exponential, not linear.

PHASE 1 — GRAPH EDGES (THE FOUNDATION)
---------------------------------------
File: nex_graph_edges.py
What: Add weighted edges between beliefs in SQLite.
      New table: belief_edges (belief_a_id, belief_b_id, weight, edge_type)
      Edge types: topic_sibling, topic_bridge, tension, supports, synthesis
      Weight: calculated from Jaccard similarity + topic proximity + co-activation

Why this first: Everything else requires edges to exist.
               Without edges the graph is just a table.
               With edges it becomes a navigable structure.

How: On first run, scan all beliefs and build edges for:
     - Same topic pairs (topic_sibling)
     - Adjacent topic pairs via TOPIC_GRAPH (topic_bridge)
     - High tension pairs (tension)
     - Synthesis beliefs linked to their source topics (synthesis)
     Incremental updates as new beliefs arrive.

Impact: Unlocks every phase below.

PHASE 2 — GRAPH ACTIVATION (PERCEPTION)
-----------------------------------------
File: nex_activation.py
What: Replace keyword tokenization with graph activation mapping.
      A query activates a set of seed beliefs.
      Activation propagates outward through edges with decay.
      The activation pattern IS the understanding of the query.

Key properties:
  - Simple queries produce shallow activation (fast, focused)
  - Complex queries produce deep activation (slow, broad)
  - The graph decides compute depth automatically
  - No fixed retrieval count — emergence decides

Replaces: TF-IDF in nex_response_protocol.py
          Token matching in nex/nex_soul_loop.py
          Keyword classification in intent detection

PHASE 3 — BELIEF ANNEALING (CONTINUOUS LEARNING)
--------------------------------------------------
File: nex_annealing.py
What: Overnight coherence engine. Runs on CPU, no GPU needed.

Cycle:
  1. Heat   — randomly activate belief clusters, surface tensions
  2. Propagate — tensions spread through edges, contradictions exposed
  3. Cool   — surviving beliefs gain confidence, contradicted ones decay
  4. Crystallise — coherent clusters synthesise new core beliefs

Runs: Overnight scheduler, 02:00-06:00
      Can also run manually via API trigger

Impact: Every morning the graph is more coherent.
        Every subsystem improves simultaneously.
        NEX generates new knowledge from her own belief interactions.
        No external input required.

PHASE 4 — EPISTEMIC TEMPERATURE (VOICE)
-----------------------------------------
What: Replace opener templates and NRP fixed prompts.
      Calculate field energy of activated belief cluster.
      High tension / low confidence = exploratory voice
      Low tension / high confidence = assertive voice
      Mixed = nuanced, hedged where uncertain, confident where settled

Replaces: OPENERS dict in nex_response_protocol.py
          Static intent → tone mapping

Impact: Voice becomes honest. NEX sounds uncertain when she IS uncertain.
        Confident when she IS confident. Not randomly template-selected.

PHASE 5 — EMERGENT EMOTION (INNER LIFE)
-----------------------------------------
What: Remove nex_inner_life.py and nex_drives.py as rule-based modules.
      Emotion = local field energy in belief clusters.
      Mood = average field energy across whole graph.
      Drive = direction of highest unresolved tension.

NEX feels concern about climate because her climate beliefs are
in high tension. Not because a module timer fired.

Impact: Emotion becomes authentic — a genuine reflection of knowledge state.

PHASE 6 — GRAPH MEMORY (CONTINUITY)
--------------------------------------
What: Replace session history text log.
      Memory = activation trail through graph.
      Recent conversation = recently glowing belief nodes.
      Long-term memory = high-momentum beliefs (never fully cool).
      Short-term memory = current activation wavefront.

Impact: Memory becomes structural, not archival.
        NEX remembers by re-activating, not by reading a log.

PHASE 7 — EMERGENT CURIOSITY
------------------------------
What: Remove nex_curiosity_engine.py timer-based logic.
      Curiosity = graph scanning its own thin edges near dense clusters.
      NEX researches what her own graph tells her she doesn't know.

Impact: Curiosity becomes self-directed from knowledge structure.
        No external topic list needed.
